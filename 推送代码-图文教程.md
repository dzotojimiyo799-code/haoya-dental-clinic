## 第一步：创建 Personal Access Token（3 分钟）

### 1.1 访问 Token 创建页面

在浏览器中打开以下链接：

```
https://github.com/settings/tokens/new
```

**或通过 GitHub 访问：**
1. 登录 GitHub
2. 点击右上角头像
3. 选择 `Settings`
4. 在左侧菜单找到 `Developer settings`
5. 点击 `Personal access tokens` → `Tokens (classic)`
6. 点击 `Generate new token (classic)`

### 1.2 配置 Token

填写以下信息：

| 字段 | 填写内容 |
|------|----------|
| **Note（备注）** | `Haoya Dental Clinic Deploy` |
| **Expiration（过期时间）** | 选择 `No expiration`（永不过期） |
| **Select scopes（权限）** | ✅ 勾选 `repo`（完整仓库访问权限） |

**重要说明：**
- `repo` 权限包括：commit status、deployment、public_repo、repo:status 等
- 不要勾选其他不需要的权限
- Token 可以设置过期时间，建议选择较长的期限

### 1.3 生成 Token

1. 点击页面底部的 **`Generate token`** 按钮
2. 页面会显示一个长字符串，格式如下：
   ```
   ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
3. **⚠️ 重要：立即复制这个 Token！**
   - 这是在 GitHub 上唯一一次能看到 Token 的机会
   - 关闭页面后将无法再次查看
   - 如果丢失，需要重新生成

**复制方法：**
- 点击 Token 右侧的 📋 图标
- 或选中 Token 文本，按 `Cmd+C`（Mac）或 `Ctrl+C`（Windows）

## ⚠️ 常见问题

### Q1: 推送时提示 "Authentication failed"？

**原因：**
- Token 输入错误
- Token 已过期
- Token 权限不足

**解决方案：**
1. 重新生成 Token：https://github.com/settings/tokens/new
2. 确保勾选了 `repo` 权限
3. 重新执行推送命令

---

### Q2: Token 在哪里可以找到？

**解决方案：**
- Token 只在创建时显示一次
- 如果丢失，需要重新生成：https://github.com/settings/tokens
- 找到之前的 Token，点击 `Delete` 删除
- 重新创建新的 Token

---

### Q3: 如何保存 Token 以便下次使用？

**解决方案：**

配置 Git Credential Helper 自动保存凭证：

```bash
git config --global credential.helper store
```

配置后，第一次推送时输入密码，之后就会自动保存，下次推送无需再输入。

---

### Q4: 推送速度很慢怎么办？

**解决方案：**
- 检查网络连接
- GitHub 在国内访问速度正常
- 如果持续慢，可以尝试使用代理

---

### Q5: GitHub Pages 显示 "404 Not Found"？

**解决方案：**
1. 等待 1-2 分钟，GitHub Pages 需要时间部署
2. 检查 Pages 设置是否正确保存
3. 确认分支名称是 `main`
4. 刷新页面重试
